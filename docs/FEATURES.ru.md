# Athena — справочник функций (RU)

> English version: [FEATURES.en.md](FEATURES.en.md)

Athena — переносимая агентная ОС: одна команда поднимает всё личное агентное окружение
на чистом Mac. Документ подробно описывает каждую функцию.

---

## 1. Слоистый bootstrap (`bootstrap.sh`)
**Что:** Единый оркестратор, собирающий систему упорядоченными слоями (0, 0b, 1, 1b, 2–6).
**Как:** Каждый слой — идемпотентная shell-функция; `--only=<N>` гонит один срез, `--dry-run`
печатает действия без исполнения. Все пути — от `$HOME`, без хардкода `/Users/...`.
**Зачем:** Воспроизводимость. Повтор не дублирует; чистый Mac за один прогон → живая система.

## 2. Merged-source (`generic ⊕ приватный overlay`)
**Что:** Источник дотфайлов собирается из публичного generic-канона (`./chezmoi`) плюс
опциональный приватный overlay (`athena-private`).
**Как:** Слой 1 rsync'ит generic (`--delete` для чистой базы), накладывает приватный (overlay
побеждает на конфликте), затем один `chezmoi init --apply`. Без overlay — generic-only.
**Зачем:** Один публичный каркас + один приватный слой = ноль секретов в публичном репо, полная система владельцу.

## 3. Идемпотентность и переносимость
**Что:** Любая операция безопасна к повтору и без машино-специфичных значений.
**Как:** Защищённые клоны (`[ -d .git ]`), пути от `$HOME`, личное — в `athena.config.sh`
(gitignored) / Keychain / chezmoi data.
**Зачем:** Один и тот же репо разворачивается на любом Mac и под любым юзером.

## 4. Слой 0 — база
**Что:** Базлайн Homebrew + CLI.
**Как:** Ставит Xcode CLT, Homebrew, затем `brew bundle` из `Brewfile`.
**Зачем:** Гарантирует тулчейн, на который опираются остальные слои.

## 5. Слой 0b — инструменты (`~/tools`)
**Что:** Внешние инструменты (напр. боты), клонируемые **до** Сознания.
**Как:** Читает `tools.manifest` (`<git-url> <путь> [install-команда]`), клон в `~/tools/*`.
**Зачем:** Идёт до Слоя 1 — chezmoi `run_once_` сможет настроить `.env` бота при apply.

## 6. Слой 1 — Сознание (дотфайлы / канон агент-ОС)
**Что:** Плоскость конфигурации агент-ОС: `~/.claude`, `~/.codex`, `~/.agents`.
**Как:** Рендерит merged chezmoi-source — конституция (`CLAUDE.md`), rules, hooks, agents,
skills-канон, реестр SSOT.
**Зачем:** Консистентный версионируемый агентный мозг, который ездит с тобой.

## 7. Слой 1b — плагины
**Что:** Переустановка маркетплейсов и плагинов.
**Как:** Читает `plugins.manifest`; `claude plugin marketplace add` / `plugin install`.
**Зачем:** Плагины декларативны, не ставятся вручную на каждой машине.

## 8. Слой 2 — реестр SSOT
**Что:** Реестр способностей (единый источник истины skills/MCP/agents).
**Как:** Пересборка `build_registry.py` / `build_views.py` / `validate.py` в `~/.agents/registry`.
**Зачем:** Graph-first маршрутизация способностей требует свежего валидного реестра.

## 9. Слой 3 — проекты (`~/Проекты`)
**Что:** Клон рабочих проектов.
**Как:** Читает `projects.manifest` (приватный перекрывает дефолт); защищённые клоны + опц. install.
**Зачем:** Новая машина получает все рабочие репо в каноническом доме.

## 10. Слой 4 — Мозг (vault знаний, `~/Мозг`)
**Что:** Vault durable-знаний (метод Карпаты «второй мозг»).
**Как:** Клон `ATHENA_VAULT_REPO` в `~/Мозг`; синтез-на-записи, hot/cold секции.
**Зачем:** Знание компаундит, а не переоткрывается каждую сессию.

## 11. Слой 5 — секреты · MCP · launchd (fail-closed)
**Что:** Рантайм: `~/.secrets` (700), напоминание о MCP-реавторизации, регистрация launchd-агентов.
**Как:** launchd через `launchctl bootout` → `bootstrap gui/$UID` (не deprecated load/unload),
счётчик `loaded/errs`, `ok` **только** при `errs==0`, агрегированный ненулевой exit при сбое.
**Зачем:** Никаких тихих «зелёных, но мёртвых» агентов — сигнал прогона честный.

## 12. Слой 6 — smoke
**Что:** Проверки инвариантов в конце прогона.
**Как:** Гоняет `smoke/smoke.sh` (см. №13).
**Зачем:** Быстрая уверенность, что структура и паритет держатся.

## 13. Smoke-гейты (`smoke/smoke.sh`)
**Что:** Шесть семейств инвариантов: структура, чистота путей, credential-shaped токены,
**личные данные**, наличие канона, наличие self-learning, паритет Claude↔Codex.
**Как:** `grep`-гейты — источник истины; гейт личных данных даёт **RED** на любое имя владельца /
username / приватный идентификатор (доказано инъекционным тестом).
**Зачем:** Регрессии (утёкшее имя, пропавший файл канона) валятся громко и автоматически.

## 14. Clean-room dry-validate (`smoke/dry-validate.sh`)
**Что:** Эмулирует рендер chezmoi без установки chezmoi.
**Как:** Собирает merged-source в temp, подставляет известные vars, валидирует плисты (`plutil`),
JSON (settings), синтаксис скриптов, ловит неизвестные `{{ }}`. Generic-only = GREEN; отсутствие
**ожидаемого** overlay (`ATHENA_EXPECT_OVERLAY=1`) = RED.
**Зачем:** Типобезопасность шаблонов на каждом прогоне; зелень не врёт о пути публичного клона.

## 15. Security guard (deny-щит)
**Что:** Детерминированный хук-слой + `permissions.deny` в `settings.json`.
**Как:** `hooks/security-guard.sh` режет запись в чувствительные пути; deny-лист покрывает
опасные паттерны — защита кодом, не текстом.
**Зачем:** Единый защитный слой едет вместе с дотфайлами.

## 16. Подсистема самообучения (`skills/self-learning`)
**Что:** Переносимая петля обучения на чекпойнтах / в конце сессии.
**Как:** Скилл `self-learning` ведёт четыре шага — (1) ошибка → корневая причина → фикс →
**системное правило**; (2) token-аудит → экономия в следующий раз; (3) данные о владельце +
повторяющиеся паттерны → память/автоматизация; (4) routing-исход. Append-only логи
(`~/.claude/self-learning/`, создаются раз, не затираются) + ретро-шаблон. Метод Карпаты:
синтез-на-записи, «одна метрика делает петлю умной», keep-or-revert для автономных петель.
**Зачем:** Каждая сессия компаундит в durable-знание, переносимое с ОС.

## 17. Жизненный цикл сессии
**Что:** Закрытие по чекпойнту, continuity через память.
**Как:** При CTX ≥ 80% или логическом чекпойнте: самообучение → компактный хэндофф →
закрытие сессии. Жнец сессий освобождает зависшие.
**Зачем:** Раздутый контекст — главная утечка токенов; continuity в памяти, не в длинном контексте.

## 18. CI (`.github/workflows/smoke.yml`)
**Что:** Непрерывная верификация на push/PR.
**Как:** macOS-раннер ставит shellcheck, гоняет `smoke.sh` и clean-room generic-only dry-validate.
**Зачем:** Гейты честности гоняются на каждое изменение, не только локально.
