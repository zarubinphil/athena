# Knowledge Wiki — конституция (метод Карпаты)

> Лежит в корне vault. Claude видит её каждую сессию (`cd <vault> && claude`).
> Импортит рой: `@AGENTS.md`. Личное/секреты — НЕ здесь.

## Структура
- `raw/` — источники, **immutable**, не редактировать.
- `wiki/` (= тематические разделы) — **ты владеешь**: синтезируешь, перезаписываешь.
- `outputs/` — ответы, отчёты.
- `_МАСТЕР-ИНДЕКС.md` — index: страница → 1 строка summary. **Читать ПЕРВЫМ** (вместо vector DB).
- `_ROUTING.md` — триггер → раздел, совпадение ≥70%.
- `Лог обработки.md` — append-only журнал ingest.

## Как ты думаешь
- **Синтез на ЗАПИСИ, не на запросе.** RAG не используем — знание скомпилировано в вики один раз.
- Cold-знание грузить через индекс **on-demand**, не всё сразу.
- **Ingest:** парсь сущности/концепты → обнови несколько wiki-страниц → поставь кросс-ссылки `[[...]]` (12+) → обнови `_МАСТЕР-ИНДЕКС.md` → допиши `Лог обработки.md`.
- **Weekly health-check:** ищи противоречия, страницы-сироты, недостающие концепт-страницы.
- Дубли не плодить — при ingest сверять со ВСЕМ vault, не с недавними нотами.

## Границы
- Не трогать `raw/`. Секреты не писать в ноты. Сортировщик физически `mv`, не «предлагает».

## Модель
kb-ingest/router → Haiku/Sonnet (bulk). kb-synth → Opus (спорный синтез). Выверять под задачу.
