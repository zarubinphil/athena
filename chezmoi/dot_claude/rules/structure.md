# Конституция раскладки

> Декларатив «что куда» — здесь и в skill `organize`. Инвариант «куда НЕЛЬЗЯ писать» — в PreToolUse-hook (детерминированно, не текстом). Процедура «разложи это» — skill `organize`.
> Устаревшая конвенция ХУЖЕ отсутствующей — агент исполняет её уверенно. Stale → архивировать, не оставлять активной.

## 1. Три плоскости — не смешивать

| Плоскость | Канон-дом | Что туда | Чем управляется |
|---|---|---|---|
| **Мозг** (агент-ОС) | `~/.claude` · `~/.codex` · `~/.agents` | конституция, rules, hooks, agents, skills-канон, реестр SSOT | chezmoi (личный канон) |
| **Знания** | `~/Полезные знания` (vault) | durable-знание по методу Карпаты, секции `NN Название` | приватный репо `vault-znaniya` |
| **Работа** | `~/Проекты/<latin-kebab>` (код) · `~/Хранилище` (жизнь) · `~/Архив` (холод) | проекты, документы, медиа | per-project git + install.sh |

Секрет, медиа, код, заметка — каждое в свою плоскость. Рантайм-мусор (сессии/кеш/телеметрия) → `_runtime/`, не в канон.

## 2. Именование

- Код-проекты: `latin-kebab` (`athena-os`, `coffee-bar`) — избегаем NFD-Unicode багов в путях скриптов.
- Секции vault: `NN Название` двузначным префиксом (Johnny-Decimal-стиль), канон — `99 Система/00 СИСТЕМА — Агентный конвейер знаний.md`.
- Архив: префикс даты `разобрано-ГГГГ-ММ/`.
- Компоненты/типы — по правилам стека проекта (см. его `rules/`).

## 3. Один канон-дом на тему

Каждая тема живёт в ОДНОМ месте. Всё прочее → `~/Архив/` с файлом-указателем `→ канон тут`. Не «ещё копия про запас». Дубль обнаружен → слить в канон, остальное в архив.

## 4. Одна воронка интейка

Только `~/Desktop/_ВХОДЯЩИЕ/` → Мнемозина → vault. Источники после обработки → `_ВХОДЯЩИЕ/_archive/ГГГГ-ММ/` (durable). Других инбоксов не плодить.

## 5. INDEX.md на папку (рост без каши)

Каждая значимая папка несёт `INDEX.md` (1 строка на элемент). CLAUDE.md/конституция = роутер по INDEX-ам, не склад. Монолитные корпуса резать по осям (context/agents/prompts/evals), иначе слабеют Graphify-связи.

## 6. Таблица размещения (дано → куда)

| Дано | Плоскость | Куда |
|---|---|---|
| новый код-проект | Работа | `~/Проекты/<kebab>/` из claude-starter (specs/ src/ evals/ + business/ ai-clone/ plans/ + INDEX.md) |
| новое durable-знание | Знания | секция vault по `_ROUTING.md`, нота с `source:` + «как поможет мне» |
| входящий материал/URL | интейк | `~/Desktop/_ВХОДЯЩИЕ/` → пайплайн |
| личный документ/счёт/медиа | Работа/жизнь | `~/Хранилище/{Документы,Финансы,Медиа,Личное}` |
| секрет/ключ/.env/.ovpn | Мозг | `~/.secrets/` (chmod 700) или Keychain — НИКОГДА в код/git |
| новое правило-всегда | Мозг | `~/.claude/CLAUDE.md` (факт) или `rules/` (с `paths:`) |
| пошаговый workflow | Мозг | skill (`skills/<name>/SKILL.md`) |
| обязательная проверка | Мозг | hook + `settings.json` |
| устаревшее/дубль | Архив | `~/Архив/` + указатель |

## 7. Пороги роста

- **Новая секция vault** — только если материал не ложится в существующую ≥70%, это устойчивая область жизни, ожидается ≥2 знания. Иначе — в ближайшую с тегом.
- **Новый проект** — отдельный репо, когда у работы свой жизненный цикл/деплой. Иначе — папка внутри существующего.
- **Вынести фишку в policy** — когда повторяется (повторная ошибка/требование). Метрика зрелости: ноль повторных «нет, не так» за неделю.
- «Не открывалось 6 месяцев» → кандидат в архив.

## 8. Граница skill ↔ policy ↔ hook

- Декларатив «что куда» → этот файл + skill `organize`.
- Деструктив / инвариант «куда нельзя» → PreToolUse-hook (детерминированно).
- Внешнее действие (деплой/публикация) → skill с `disable-model-invocation: true`.
- CLAUDE.md и skill — advisory, исполняются вероятностно. Жёсткое — только hook/`permissions`.

## 9. Свежесть (анти-дрейф)

Это правило ревизуется на каждом ретро. Конфликт с `~/.claude/CLAUDE.md`/`AGENTS.md`/hooks → чинить немедленно. Меняешь раскладку — обнови `organize` skill и bootstrap-project/gsd/kb-classify в локстеп, иначе рост пойдёт по старому.
