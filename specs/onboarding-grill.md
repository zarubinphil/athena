# Спец: грилл первого запуска (onboarding-grill)

> Статус: **дизайн утверждён** (Филипп, 2026-06-16). Код — после P0-прелонч-гейта.
> Тип: расширение skill `setup-os`. Public-safe (личных значений в репо нет).
> Метод: брейнсторм → этот спец → writing-plans → реализация.

## 1. Цель

На **первом запуске** Athena OS на новой машине новый пользователь проходит **полный грилл-опрос** (стиль `/grill-me`), который собирает его предпочтения, личные данные и операционные правила и из них **идеально** составляет `owner.md`, `CLAUDE.md`, `AGENTS.md`, манифесты и `references/*`. «Идеально» = по методу Карпаты + дистилляту best-practice, зашитому в репо как generic-рубрик.

Проблема сейчас: `setup-os` Этап A — мелкий (5 вопросов → owner.md + копия статичного CLAUDE.template). Нет авто-триггера, нет глубины, метод Карпаты опционален.

## 2. Скоуп

**В скоупе:**
- SessionStart-детектор первого запуска + nudge (хук) → грилл ведёт Claude.
- Полный грилл P1–P5 (обязателен, чекпойнт-able), банк вопросов как asset.
- Карпаты-принципы — **mandatory**-секция, не опциональный этап.
- `CLAUDE.md` строится **на базе существующего** `CLAUDE.template.md` (= структура конституции владельца), грилл персонализирует через hot/cold-роутинг (глубина → `references/`, не раздувать CLAUDE.md).
- Рубрик композиции + банк вопросов + детальный метод Карпаты как baked assets.

**Вне скоупа:**
- Личные значения владельца в публичном репо (P0.2 sanitize это и запрещает).
- Хранение значений секретов (только МАП — где лежит).
- Переписывание архитектуры bootstrap.sh (вставляем чисто, не трогаем слои).

## 4. Архитектура — 5 дельт

1. **Детектор+триггер** (новое). SessionStart-хук `chezmoi/dot_claude/hooks/onboarding-detect.sh` (chezmoi-шаблон). Хук НЕ интервьюирует (shell неинтерактивен) — детектит «не персонализировано» и инжектит `additionalContext`: «Athena не настроена → запусти setup-os». Грилл ведёт Claude через AskUserQuestion.
2. **Полный грилл** (расширить Этап A). Заменить single-screen на структуру P1–P5, управляемую `assets/interview-rubric.md`. Чекпойнт/резюм через прогресс-стамп.
3. **Динамический hot/cold-роутинг** (улучшить запись). Грилл маршрутизирует ответы: hot → `CLAUDE.md`/`owner.md`, cold → `references/*`. CLAUDE.md остаётся лин (≤200), глубина в references. CLAUDE.md = база `CLAUDE.template` + точечные слоты + указатели.
4. **Карпаты mandatory** (поднять Этап B). Vault-сетап + принципы синтеза — обязательная часть онбординга, не «можно пропустить». Создать `references/karpathy-method.md` (детальный метод). Грилл обязан поставить: vault (raw/wiki/outputs + _МАСТЕР-ИНДЕКС + _ROUTING + Лог + vault-CLAUDE + AGENTS-рой), синтез-на-записи, hot/cold, граф-линки `[[]]`.
5. **Baked рубрик** (новые assets). `assets/interview-rubric.md` (банк вопросов P1–P5), `assets/composition-principles.md` (как идеально писать файлы). Все generic, public-safe.

## 5. Полный грилл — фазы P1–P5

| Фаза | Собирает (20 доменов сгруппированы) | Пишет | Tier |
|---|---|---|---|
| **P1 Идентичность** | роль/тип занятости, язык, локация+семья-ограничения, домены экспертизы | `owner.md` | hot |
| **P2 Проекты+стек** | активные проекты (имя/статус/приоритет), tech-стек, инструменты к установке | `owner.md`, `projects.manifest`, `tools.manifest` | hot |
| **P3 Стиль+правила** | model-routing стенс (цена↔качество), коммуникация (тон/вербозность/caveman?), структура ответа, правила НИКОГДА/ВСЕГДА, gate на пакеты | `CLAUDE.md` (слоты) | hot |
| **P4 Операц-преференсы** | security-постура (поверхности риска), design/UX-вкус (+анти-паттерны), deploy/сервер, swarm/agent-routing, web-каскад | `references/{security,design,deploy,agents,scrapegraph}.md` | cold |
| **P5 Инфра** | vault-сетап (Карпаты), session-lifecycle/токен-дисциплина, self-learning логи, **secrets-МАП (не значения)** | `CLAUDE.md`+`references/`+`secrets-map.md`+`vault/` | cold |

**Поток фазы** (наследует принцип setup-os): объясни 1 фразой → AskUserQuestion (батч по домену) → **синтезируй** ответ (не дамп сырья — Карпаты синтез-на-записи) → запиши в целевой файл → покажи → «дальше?».

**Чекпойнт/резюм:** после каждой фазы — стамп прогресса (напр. строка в `~/.claude/.athena-onboarding-progress`). «Полный грилл» НЕ значит «один присест» — значит полное покрытие; юзер может паузить и резюмить с последней фазы.

**Fast-path** = только структура чекпойнта (P1+P2-минимум → рабочий конфиг, дальше резюм). Не отдельный режим — точка возобновления.

**Gate полноты (Карпаты):** каждый домен P1–P5 либо заполнен, либо явно пропущен-с-причиной (залогировано). Нет тихих дыр в конфиге — зеркало completeness-gate vault.

## 6. Принципы Карпаты — ОБЯЗАТЕЛЬНЫ

Грилл и генерируемые файлы обязаны воплощать (детали → `assets/composition-principles.md` + `references/karpathy-method.md`):

1. **Синтез на ЗАПИСИ, не на запросе.** Ответы юзера синтезируются в атомарный конфиг сразу, не складируются сырьём. RAG не используем.
2. **Hot/cold split.** Hot (цель/состояние/правила) → `CLAUDE.md`/`owner.md`. Cold (глубина/история/SOP) → `references/`/skills/vault. Грузить on-demand через указатель.
3. **Мастер-индекс первым.** Vault получает `_МАСТЕР-ИНДЕКС.md` (страница→1 строка) + `_ROUTING.md` (триггер→раздел ≥70%). Читается до сканирования корпуса.
4. **Атомарность + `source:`.** Каждая wiki-нота: title/type/status/**source**/verification/`[[links]]`/применимость. Без source — нота не валидна.
5. **Граф-линки `[[]]` first-class.** Каждый ответ/нота линкуется к разделу, не висит сиротой. `/graphify` строит граф.
6. **Append-only лог.** `Лог обработки.md` — журнал решений ingest, для аудита/самокоррекции.
7. **Onboard-once.** Один верный конфиг → агент работает корректно во ВСЕХ будущих сессиях. Конфиг самодостаточен.
8. **Weekly health-check.** Противоречия/сироты/пробелы — на каденции (cron-агент, Этап E).

## 7. CLAUDE.md на базе твоего

`CLAUDE.template.md` уже = сани́тайзнутая структура конституции владельца. Грилл НЕ переписывает её генеративно (риск дрейфа) — **анкорит к проверенному скелету** + персонализирует точечно:
- Копирует `CLAUDE.template.md` как базу.
- Заполняет малые hot-слоты из P3: дефолт-язык, model-routing-стенс (1 строка), доп. НИКОГДА/ВСЕГДА (если юзер задал свои).
- Глубина (P4/P5) НЕ инлайнится — уходит в `references/*`, CLAUDE.md лишь указывает (`@references/...`). Держит ≤200 строк (instruction-budget).
- owner.md (P1/P2) подключается строкой `@references/owner.md`.

Итог: CLAUDE.md каждого юзера = структура владельца (проверенная) + его личные значения, лин и hot/cold-корректный.

## 9. Детект и триггер

**Сигнал «не персонализировано»** (детектор, в порядке надёжности):
1. Маркер `~/.claude/.athena-onboarded` отсутствует → fresh.
2. Фолбэк-эвристика: `owner.md` содержит `{{` placeholder-ы → не заполнен.

**Хук** (`onboarding-detect.sh`, SessionStart): если fresh → echo additionalContext-nudge. Иначе молчит. **Не нагло:** маркер ставится по завершении грилла (или по явному «позже»/«пропустить» — snooze-маркер), чтобы не долбить каждую сессию.

**Триггер интервью:** `/setup-os` (ручной, и для апдейта секций) ИЛИ nudge от хука. bootstrap.sh интервью НЕ запускает (shell неинтерактивен) — Layer 1 лишь ставит хук+шаблоны; первый `claude`-сеанс → детект → грилл. Чистая граница shell↔интерактив (`rules/structure.md §66-71`).

## 10. Жёсткие границы

- **Public ≠ private.** Assets (rubric/templates/principles) — generic, в репо. Сгенерённые файлы (owner.md, заполненный CLAUDE.md, манифесты) — личные, в `~/.claude`/overlay/gitignored, **НИКОГДА в публичный athena-os**. Совпадает с P0.2 sanitize.
- **Secrets — map-only.** Грилл собирает МАП (какие секреты есть / где лежат) → `secrets-map.md` (gitignored) + чеклист на ручное заполнение Keychain/.env. Значений грилл НЕ хранит/НЕ спрашивает-в-репо. Правило НИКОГДА.
- **Instruction-budget.** CLAUDE.md ≤200, проектный ≤150, AGENTS бюджетирован. Излишек → references/skills.
- **Idempotent.** Маркер глушит повтор; re-run мёржит-не-перетирает; `cp X X.bak` до перезаписи (один канонический дом, `structure.md §3`).

## Приложение A — Банк вопросов (interview-rubric, дистиллят)

> Источник: майнинг vault (метод Карпаты, JTBD, AI-literacy) + таксономия канон-конфига. Generic, без личных значений. Техника: **поведение-не-ярлык** (спрашивать что агент должен УМЕТЬ, не «кто ты»); покрыть 6 мер AI-literacy (Role/Task/Context/Process/Output/Verification).

**P1 Идентичность → owner.md (hot)**
- Роль и тип занятости (соло / фирма / фаундер; сколько потоков дохода)?
- Дефолт-язык общения агента?
- Личные/семейные ограничения (часы, окна доступности)?
- Домены экспертизы, где ты — авторитет, не агент?

**P2 Проекты+стек → owner.md / projects.manifest / tools.manifest (hot)**
- Активные проекты: для каждого — что это, статус, критичность?
- Дефолт tech-стек (frontend/backend/mobile/infra/AI-провайдер)?
- Какие проекты/инструменты клонировать/ставить на новой машине?

**P3 Стиль+правила → CLAUDE.md (hot)**
- Model-routing: cost-оптимизация или quality-максимизация? Когда задача оправдывает дорогую модель?
- Коммуникация: насколько терсо/вербозно? Форматы (буллеты/проза/заголовки/лимиты)? caveman?
- Структура каждого ответа (напр. «что сделано · что нужно · следующий шаг»)?
- НИКОГДА: что агент не делает никогда? ВСЕГДА: что до/после задачи обязательно?
- Gate: спрашивать перед установкой пакетов/clone/adopt? Что можно свободно?

**P4 Операц-преференсы → references/* (cold)**
- Security: какие поверхности несут реальный риск (auth/платежи/данные/инфра)? Какие проверки всегда, даже в ущерб скорости?
- Design/UX: вкус одной фразой? 2–3 референса? Анти-паттерны (что не хочешь видеть)?
- Deploy: куда деплоятся проекты? Провайдер/метод (SSH/CI/CDN)? Доступ к серверу?
- Swarm: какие паттерны → рой vs соло? Любимый фреймворк оркестрации? Что требует явного добра?
- Web-инструменты: что первым (scrape/search/crawl)? Цепочка фолбэка?

**P5 Инфра → CLAUDE.md/references/secrets-map/vault (cold)**
- Vault: ведёшь базу знаний? Где? Что триггерит добавление, как индексируется? (→ Карпаты-сетап обязателен.)
- Session-lifecycle: на каком % контекста предупреждать? Что должно пережить границу сессии? Как хэндофф?
- Self-learning: логировать ошибки/паттерны? Где, в каком формате, когда промоутить в правила?
- Secrets-МАП: категории секретов (API-ключи/SSH/DB)? Для каждой — где источник истины? **(только МАП, не значения.)**

## Приложение B — Принципы композиции (composition-principles, дистиллят)

- **Instruction-budget:** CLAUDE.md ≤200 (проектный ≤150), AGENTS бюджетирован. Длинное → skills/docs.
- **Hot/cold:** hot (цель/состояние/правила) в конституцию; cold (глубина/история) в references/skills, on-demand.
- **Поведение-не-ярлык:** элиситируй что агент должен УМЕТЬ в день 1, не персона-лейбл. (JTBD: situation→motivation→progress→workaround→tradeoffs.)
- **Onboard-once:** один верный конфиг → корректная работа во всех сессиях. Самодостаточность.
- **Two-file-minimum старт:** большинству хватает CLAUDE.md + settings.json; директории добавлять по мере стабилизации workflow. Дашборд/обсервабилити — последним.
- **AI-literacy 6-мер:** хороший набор инструкций покрывает Role / Task / Context / Process / Output / Verification.
- **5-секц AGENTS-скелет:** Setup → Standards → Checks → Watch-outs → Done. «Done» = тесты прошли + ревью + хэндофф. Checks — наблюдаемы/тестируемы, не аспирации.
- **Self-learning reference 5-часть:** (1) статический указатель (on-demand, не hot), (2) domain-map тиров (primary/secondary/specialized/excluded), (3) принципы домена, (4) линковка к правилу-триггеру конституции, (5) auto-growth хук (vault дописывает при обработке материала). Декаплит размер hot-контекста от глубины домена.
- **Living-doc:** повтор коррекции дважды → одно конкретное правило в Watch-outs. Периодически чистить стейл-команды.

## Приложение C — Метод Карпаты (karpathy-method, дистиллят)

- **Синтез на записи, не на запросе.** Vault = скомпилированная вики, не дамп для retrieval. RAG не используем.
- **Мастер-индекс дешёвым первым пасом.** `_МАСТЕР-ИНДЕКС.md` (раздел→summary→счётчик нот) читается до сканирования.
- **Hot/cold split** (как выше).
- **Атомарные ноты** с обязательными полями: title/type/status/**source**/verification/`[[links]]`/применимость.
- **Gate полноты (инвариант):** каждый inbox-item → нота с `source:` ИЛИ задокументированная причина исключения. Нет тихих дропов. Архив — только после формального gate-3.
- **Граф-линки `[[]]`** first-class; `/graphify` строит граф.
- **Append-only лог** решений обработки.
- **Weekly health-check:** противоречия/сироты/пробелы на каденции.
- **Структура vault:** `raw/` (immutable) · `wiki/` (LLM владеет, синтезирует) · `outputs/` · `_МАСТЕР-ИНДЕКС.md` · `_ROUTING.md` · `Лог обработки.md` · `CLAUDE.md` (vault-конституция) · `AGENTS.md` (рой kb-ingest/router/synth/health).
- **Лестница Prompt→Skill→Gate→System:** сырой промпт → reusable skill → approval-gate → автономный цикл. Никогда не стартовать с автономного.
