# Принципы композиции — как идеально писать конфиг

> Используется гриллом (skill `setup-os`) при синтезе `owner.md` / `CLAUDE.md` / `AGENTS.md` / `references/*`. Generic, public-safe.
>
> Цель: из ответов грилла собрать **самодостаточный** конфиг по методу Карпаты, а не дамп сырья. Синтез на записи.

## Бюджет инструкций

- `CLAUDE.md` ≤ 200 строк; проектный `CLAUDE.md` ≤ 150; `AGENTS.md` бюджетирован.
- Длинное / редко-нужное → `skills/`, `references/`, `docs/`. Конституция = роутер, не склад.
- Излишек в hot-контексте = утечка токенов каждую сессию.

## Hot / Cold split

- **Hot** (грузится каждую сессию): цель, состояние, правила НИКОГДА/ВСЕГДА, model-routing, структура ответа → `CLAUDE.md` / `owner.md`.
- **Cold** (on-demand через указатель): глубина домена, история, SOP, security/design/deploy-детали → `references/*` / skills / vault.
- В CLAUDE.md cold-тема присутствует ОДНОЙ строкой-указателем (`@references/...`), не инлайном.

## Поведение, не ярлык

- Элиситируй что агент должен УМЕТЬ в день 1, не персона-лейбл.
- JTBD-каркас: situation → motivation → progress → workaround → tradeoffs.
- «Хочу чтобы перед деплоем всегда гонял smoke» > «я аккуратный».

## Onboard-once

- Один верный конфиг → корректная работа во ВСЕХ будущих сессиях. Самодостаточность — инвариант.
- Конфиг не должен требовать допобъяснений в рантайме; всё нужное — в нём или по указателю.

## Two-file-minimum старт

- Большинству хватает `CLAUDE.md` + `settings.json` на старте.
- Директории (`rules/`, `agents/`, `references/`) добавлять по мере стабилизации workflow.
- Дашборд / обсервабилити — последними, не вперёд рабочего цикла.

## AI-literacy — 6 мер

Хороший набор инструкций покрывает: **Role · Task · Context · Process · Output · Verification**. Проверяй сгенерённый конфиг по этим 6 — дыра в любой = неполный онбординг.

## AGENTS.md — 5-секционный скелет

`Setup → Standards → Checks → Watch-outs → Done`.

- **Done** = тесты прошли + ревью + хэндофф (конкретно, не «готово»).
- **Checks** — наблюдаемы/тестируемы (команда/метрика), не аспирации.

## Self-learning reference — 5 частей

1. Статический указатель (on-demand, НЕ hot).
2. Domain-map тиров (primary / secondary / specialized / excluded).
3. Принципы домена.
4. Линковка к правилу-триггеру конституции.
5. Auto-growth хук (vault дописывает при обработке материала).

Декаплит размер hot-контекста от глубины домена.

## Living-doc

- Повтор одной коррекции дважды → одно конкретное правило в Watch-outs / НИКОГДА.
- Периодически чистить стейл-команды (устаревшая конвенция ХУЖЕ отсутствующей).

## CLAUDE.md — анкор-к-скелету, не генерация с нуля

`CLAUDE.template.md` = проверенный скелет конституции. Грилл его **не переписывает генеративно** (риск дрейфа):

- Копирует template как базу.
- Заполняет малые hot-слоты из P3 (язык, model-стенс, доп НИКОГДА/ВСЕГДА).
- Глубину (P4/P5) уводит в `references/*`, оставляя указатель.
- Подключает owner.md строкой `@references/owner.md`.
- Держит ≤ 200 строк.
