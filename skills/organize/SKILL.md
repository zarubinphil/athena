# organize — авто-раскладка по конституции

Процедура «дано X → положи в канон-место». Источник истины — [`rules/structure.md`](../../rules/structure.md). Деструктив/инвариант — в hook, не здесь.

## Шаги
1. **Определи плоскость** (Сознание / Знания / Работа) по таблице размещения `structure.md §6`.
2. **Определи канон-дом** и проверь, нет ли уже канона по этой теме (§3). Дубль → слить, не плодить.
3. **Создай по шаблону:**
   - код-проект → `/bootstrap-project` в `~/Проекты/<latin-kebab>/` (specs/ src/ evals/ + business/ ai-clone/ plans/ + INDEX.md).
   - durable-знание → `kb-pipeline` (Мнемозина) → секция vault по `_ROUTING.md`, нота с `source:` + «как поможет мне».
   - входящий материал → `~/Desktop/_ВХОДЯЩИЕ/`.
   - документ/медиа/секрет → `~/Хранилище/...` / `~/.secrets` (Keychain).
4. **Обнови индексы:** `INDEX.md` папки + мастер-индекс + реестр (если способность). Связи через `[[ ]]`.
5. **Проверь пороги роста** (§7): новый раздел/проект только при условиях; иначе — в ближайший с тегом, решение в лог.

## Правила
- Сначала проверь существующий канон — не создавай дубль «про запас».
- Имя код-проекта — `latin-kebab` (NFD-баги путей).
- Изменил саму раскладку → синхронь `structure.md` + bootstrap-project + kb-pipeline в локстеп.
- Не уверен в плоскости → спроси одним вопросом, не угадывай.

## Анти-триггеры
Не для: правки кода внутри проекта, ответа на вопрос, рефактора (это другие скиллы). Только размещение нового.
